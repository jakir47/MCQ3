### 2. LocalFileStorageService
Implements IFileStorageService:
- Resolves full path: ContentRootPath + storagePath
- Creates directory if missing
- Saves with FileStream / FileMode.Create
- Generates Guid-based filename, preserves original extension (lowercased)
- GetUrlAsync → "/" + storagePath.Replace("\\", "/")
- DeleteAsync → silently no-ops if file missing; log a warning

### 3. FileValidationResult
  record FileValidationResult(bool IsValid, string? ErrorMessage);

Validation rules (run before any file is written):
- File must not be null or empty
- Extension must match the declared MIME type (cross-check both)
- MIME type must be in the allowed list above
- File size must not exceed the limit for its MediaType
- For bulk uploads: validate all files first, collect all errors, 
  return a list of FileValidationResult (fail-fast per file, not 
  the whole batch)

### 7. MediaFileService
Constructor:
  (AppDbContext db, IFileStorageService storage, 
   ILogger<MediaFileService> logger)

#### Single upload
  Task<Result<MediaFileDto>> UploadAsync(
      IFormFile file, Guid uploadedByUserId, CancellationToken ct)

  - Validate → return Result.Failure if invalid
  - Determine MediaType from MIME type
  - Call storage.UploadAsync → storagePath
  - Persist MediaFile entity
  - On DB failure: call storage.DeleteAsync (compensate), return Failure
  - Return Result.Success(MediaFileDto)

#### Bulk upload
  Task<BulkUploadResultDto> UploadBulkAsync(
      IReadOnlyList<IFormFile> files, Guid uploadedByUserId, 
      CancellationToken ct)

  - Validate all files first; collect failures per file
  - For each valid file: attempt upload + DB persist independently
  - A single file failure must NOT abort the rest of the batch
  - Compensate (delete from disk) on any per-file DB failure
  - Return BulkUploadResultDto with Succeeded + Failed lists

#### Get single
  Task<Result<MediaFileDto>> GetByIdAsync(Guid id, CancellationToken ct)
  - Respects global soft-delete query filter
  - 404-style Failure if not found

#### Get many (paged)
  Task<PagedResult<MediaFileDto>> GetPagedAsync(
      MediaType? filterByType, int page, int pageSize, 
      CancellationToken ct)
  - Default page size 20, max 100
  - Ordered by CreatedAt descending
  - Respects soft-delete filter

#### Soft delete
  Task<Result> SoftDeleteAsync(
      Guid id, Guid deletedByUserId, CancellationToken ct)
  - Set IsDeleted = true, DeletedAt = UtcNow, DeletedByUserId
  - Do NOT touch the physical file
  - Return Result.Failure if already soft-deleted or not found

#### Restore (undo soft delete)
  Task<Result<MediaFileDto>> RestoreAsync(
      Guid id, CancellationToken ct)
  - Must call .IgnoreQueryFilters() to find soft-deleted records
  - Clear IsDeleted, DeletedAt, DeletedByUserId
  - Return Result.Failure if record is not soft-deleted

#### Hard delete (permanent)
  Task<Result> HardDeleteAsync(Guid id, CancellationToken ct)
  - Must call .IgnoreQueryFilters()
  - Delete physical file from storage first
  - Remove DB record
  - Use case: admin purge of soft-deleted files only; 
    reject hard-delete of non-soft-deleted records with a 
    descriptive error ("Soft-delete first before purging.")

#### Update alt text
  Task<Result<MediaFileDto>> UpdateAltTextAsync(
      Guid id, string? altText, CancellationToken ct)

### 8. PagedResult<T>
  record PagedResult<T>(
      IReadOnlyList<T> Items,
      int              Page,
      int              PageSize,
      int              TotalCount
  )
  {
      public int TotalPages => (int)Math.Ceiling(TotalCount / (double)PageSize);
      public bool HasNext   => Page < TotalPages;
      public bool HasPrev   => Page > 1;
  }
